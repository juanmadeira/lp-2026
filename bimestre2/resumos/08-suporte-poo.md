# RESUMO - Suporte à programação orientada a objetos
## Questões de projeto
* **Exclusividade**: Decisão entre linguagens "puras" onde tudo é objeto (Smalltalk, Ruby) vs. "híbridas" (Java, C++) [52-54].
* **Alocação**: Objetos podem ser estáticos ou, mais comumente, alocados no **monte (heap)** dinamicamente [53].
* **Classes Aninhadas**: Permite declarar classes dentro de outras para organizar escopo e visibilidade de recursos específicos [55].

## Tipagem e Vinculação
* **Subtipos**: Uma subclasse é um subtipo se mantiver o relacionamento "é-um" de forma compatível com a classe pai [56].
* **Verificação**: Polimorfismo pode exigir checagem de tipo dinâmica, o que é mais custoso que a estática feita no compilador [57].
