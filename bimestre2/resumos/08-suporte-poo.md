# RESUMO - Suporte à programação orientada a objetos
## Questões de projeto
* **Exclusividade**: Decisão entre linguagens "puras" onde tudo é objeto (Smalltalk, Ruby) vs. "híbridas" (Java, C++).
* **Alocação**: Objetos podem ser estáticos ou, mais comumente, alocados no monte (_heap_) dinamicamente.
* **Classes Aninhadas**: Permite declarar classes dentro de outras para organizar escopo e visibilidade de recursos específicos.

## Tipagem e Vinculação
* **Subtipos**: Uma subclasse é um subtipo se mantiver o relacionamento "é-um" de forma compatível com a classe pai.
* **Verificação**: Polimorfismo pode exigir checagem de tipo dinâmica, o que é mais custoso que a estática feita no compilador.
