# RESUMO - Aspectos preliminares sobre liguagens de programação
## Domínio de programação
### Motivo de estudo de linguagem
1. Capacidade Aumentada para expressar ideias
2. Embasamento de escolha
3. Habilidade aumentada para aprender novas linguagens
4. Melhor entendimento da importância da implementação
5. Melhor uso das linguagens já conhecidas
6. Avanço geral da computação

### Aplicações científicas
Estruturas de dados simples, mas requerem diversas computações de ARITMÉTICA.

Exemplos:
- FORTRAN (1957)
- Assembly (1951)

### Aplicações empresariais
Facilidade de elaborar RELATÓRIOS, armazenar números DECIMAIS e dados de CARACTERES.
Atualmente, planilhas gerenciadoras e sistemas de bancos de dados ocupam esse espaço.

Exemplo:
- COBOL (1960)

### Inteligência artificial
Uso de COMPUTAÇÕES SIMBÓLICAS, MANIPULAÇÃO FLEXÍVEL de informações, mesmo sacrificando eficiência.

Exemplos:
- LISP (1959)
- PROLOG (1972)

### Programação de sistemas
Eficiência na execução por propiciar suporte na execução de outros aplicativos e na criação de interface com dispositivos externos.

Exemplo:
- C (1972)

### Software para web
Exemplos:
- XHTML
- Java
- C#
- PHP

## Critério de avaliação de linguagens
### Legibilidade
Facilidade com a qual programas podem ser lidos e entendidos. Nos anos 70, codificação perde espaço e manutenção é considerada parte principal do ciclo de vida de um software.

Auxilia na leitura de um código:
- Simplicidade geral
    - Multiplicidade de recursos (i += 1, i++...), sobrecarga de operadores (+ para int, float, string...)
- Ortogonalidade
    - Uma linguagem é ortogonal quando cada recurso dela funciona de forma independente, e qualquer combinação desses recursos é valida e previsível.
- Tipos de dados
    - Utilização de mecanismos adequados para definição de tipos e estruturas de dados (1, true; 0, false...)
- Projeto da sintaxe
    - Palavras especiais, limitações na nomenclatura de variáveis etc.

### Facilidade de escrita
Mede a facilidade de uma linguagem em criar programas.

Através de:
- Simplicidade e ortogonalidade
    - Muita ortogonalidade pode prejudicar a facilidade de escrita (erros passando despercebidos)
- Suporte à abstração
    - Permitir que detalhes sejam ignorados
- Expressividade
    - Formas convenientes de expressar uma computação em que costuma representar muitos passos (cout do C++ em vez de System.out.println() do Java)

### Confiabilidade
Recursos de linguagens com efeitos na confiabilidade:

- Verificação de tipos
    - Execução de testes feitos pelo compilador ou em tempo de execução
- Tratamento de exceções
    - Interceptar erros em tempo de execução e tomar medidas corretivas
- Utilização de apelidos
    - Métodos ou nomes distintos referenciando a mesma área de memória
- Legibilidade e facilidade de escrita
    - Quanto mais fácil de escrever um programa, maior a probabilidade de que ele esteja correto
- Custo
    - De treinamento, escrita, sistema de implementação, do projeto da linguagem, de compilação, de má confiabilidade, de manutenção...
